# `.env` setup guide

This project reads configuration from environment variables (optionally via a `.env` file).

> Good news: **everything has a default**, so you can start with a minimal file and only override what you need.

## 1) Minimal `.env` (recommended start)

```dotenv
# --- Core bot settings ---
INSTRUMENTS_JSON=EUR_USD,USD_JPY,GBP_USD
GRANULARITY=M5
MIN_SIGNAL_SCORE=0.15
MAX_SPREAD_PCTILE=85
MAX_POSITIONS=6
RISK_BUDGET_DAILY=0.015
CLUSTER_RISK_CAP=0.01
MIN_TRADE_INTERVAL_SEC=5
MAX_TRADE_INTERVAL_SEC=120

# --- Optional monitoring DB/cache ---
MONITORING_EVENTS_DB_URL=sqlite:///research_outputs/monitoring/events.db
MONITORING_CACHE_SIZE=2000

# --- Dashboard (if you run ui_dashboard.py) ---
UI_HOST=0.0.0.0
UI_PORT=8000
UI_DEBUG=false
UI_BASE_PATH=
AUTO_REFRESH_SECONDS=10
MAX_LOG_BYTES=200000
LOG_LINES=200
LOG_GLOBS=*.log,*.jsonl,control_plane_logs/*.jsonl,research_outputs/*.json
PROCESS_FILTER=
```

## 3) Boolean value format

For all boolean flags, these are treated as true values:
- `1`, `true`, `yes`, `on` (case-insensitive)

Anything else is interpreted as false.
